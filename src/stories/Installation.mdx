# Installation

Package installation, peer dependencies, and Tailwind v4 integration.

---

## 1. Install the package

```bash
npm install eglador-ui-react-typewriter
```

```bash
yarn add eglador-ui-react-typewriter
```

```bash
pnpm add eglador-ui-react-typewriter
```

## 3. Add `@source` for Tailwind v4

Tailwind needs to scan classes inside the package. Add the `@source` directive to your global stylesheet:

```css
@import "tailwindcss";

@source "../node_modules/eglador-ui-react-typewriter";
```

The `@source` path varies based on your CSS file location:

## 4. Cursor blink keyframes

The cursor's blink animation uses two CSS keyframes (`eglador-tw-blink-smooth`, `eglador-tw-blink-hard`). They're injected into `<head>` automatically the first time the component mounts — **no extra CSS import needed**.

If you render the cursor in a portal or shadow DOM and need to inject the styles manually:

```tsx
import { ensureCursorStyles } from "eglador-ui-react-typewriter";

ensureCursorStyles();
```

Calling more than once is a no-op.

---

## 5. Server components note (Next.js)

The component uses `useState` / `useEffect` and is marked `"use client"`. Render it inside a client component:

```tsx
// app/page.tsx
import { Hero } from "./hero";

export default function Page() {
  return <Hero />;
}
```

```tsx
// app/hero.tsx
"use client";

import { Typewriter } from "eglador-ui-react-typewriter";

export function Hero() {
  return (
    <h1 className="text-5xl font-bold">
      We{" "}
      <Typewriter>
        <Typewriter.Item>build.</Typewriter.Item>
        <Typewriter.Item>design.</Typewriter.Item>
        <Typewriter.Item>ship.</Typewriter.Item>
      </Typewriter>
    </h1>
  );
}
```

> **SSR rendering** — on the server the component renders empty (no animation possible). The animation begins on client mount. If you need static fallback content, use the render-prop API and check `phase === "idle"`.
